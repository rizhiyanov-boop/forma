# Heatmap нагрузки + XP-система

Статус: **проектирование**, реализация после ProgressionEngine + тестов.
Дата фиксации: апрель 2026.

## 1. Концепция

Две связанные фичи: **визуализация нагрузки по группам мышц** + **уровневая система** для каждой группы. Фичи проектируются вместе потому что:
- XP-система живёт на heatmap-экране и в фигурах
- Подкраска фигур и расчёт XP идут от одной модели нагрузки

## 2. Размещение

**Комбо: мини-карточка на Home + полный экран по тапу.**

- Home: компактная карточка с 1 фигурой (только ФАКТ за неделю) + краткой строкой состояния → тап
- Полный экран «Баланс»: 2 фигуры (ПЛАН | ФАКТ) + список групп с XP/уровнями + AI-рекомендация + переключатель Перед/Зад (на бету)

## 3. Что показываем

**Две фигуры рядом, ПЛАН слева, ФАКТ справа.** Только вид спереди для альфы.

- ПЛАН = что должно быть по программе на текущую неделю
- ФАКТ = что реально сделано в завершённых сессиях этой недели

8 основных групп: Грудь, Спина, Плечи (общее), Бицепс, Трицепс, Ноги, Ягодицы, Пресс. Икры/предплечья не показываем. Дельты на пучки делим на бету.

## 4. Расчётная модель

### Единица нагрузки: рабочий сет

Не тоннаж, не повторы. Согласно RP Strength / MASS — все нормы (MEV/MAV/MRV) считаются в сетах.

Опциональные подходы — НЕ учитываем.
Разогревочные подходы (низкий вес или RIR > 4) — НЕ учитываем.
Все остальные завершённые сеты = 1 единица.

### Распределение нагрузки на группы

```
primary muscle += 1.0 за сет
secondary muscle += 0.5 за сет
```

Для альфы — фиксированное 0.5 для всех secondary. На бету — поле `engagementWeight` на упражнении (значения 0.3-0.8 в зависимости от движения).

### Шкала «100%»

**Разные шкалы на ПЛАН и ФАКТ — это фича, не баг.**

ПЛАН:
- 100% = MAV (верхняя граница оптимума по нормам RP Strength)
- < MEV → недогруз в плане (серый/жёлтый)
- MEV-MAV → оптимум (лайм)
- MAV-MRV → много, но ок (оранжевый)
- > MRV → перегруз в плане (красный)

ФАКТ:
- 100% = что в плане на эту неделю
- > 110% от плана → перевыполнение (красный, если над MRV; иначе лайм)
- 80-110% → по плану (лайм)
- 50-80% → отстаёшь (оранжевый)
- < 50% → сильно отстаёшь (жёлтый)
- 0% → не качал (серый)

Учитываем день недели — `expectedNow = plannedSets × (день+1)/7`. В понедельник «факт 0%» — норма, в воскресенье — провал.

### Цвета

Цветовая логика:
- Лайм (`AccentLime`) = оптимум / по плану — поощряем
- Оранжевый (`WarnAmber`) = много или отстаёшь
- Красный (`ErrorRed`) = перегруз
- Серый (`TextTertiary`) = недогруз / не качал
- Жёлтый (между амбер и серый) = промежуточные состояния

Принцип: **оптимум = акцентный цвет бренда**. Юзер хочет видеть лайм — это поощряемое состояние.

### MEV/MAV/MRV нормы

Базовая таблица для **продвинутого мужчины** (надо вписать в `domain/coaching/MuscleVolumeStandards.kt`):

| Группа | MEV | MAV | MRV |
|---|---|---|---|
| Грудь | 8 | 16 | 22 |
| Спина | 10 | 18 | 25 |
| Плечи | 8 | 16 | 22 |
| Бицепс | 8 | 14 | 20 |
| Трицепс | 6 | 14 | 20 |
| Квадрицепс | 8 | 14 | 20 |
| Ягодицы | 6 | 14 | 18 |
| Пресс | 0 | 14 | 25 |

Множители по уровню:
- BEGINNER × 0.6
- INTERMEDIATE × 0.85
- ADVANCED × 1.0

Пол не влияет (восстановление лучше у женщин, но MAV в литературе одинаковый).

## 5. Визуализация фигур

Подход: **стилизованная фигура из примитивов Compose** через `Canvas` (drawCircle / drawRoundRect / drawPath). 8 зон-полигонов под каждую группу мышц. Каждая зона окрашивается по своему статусу.

Если выйдет криво — fallback на «без фигуры, только bar chart» (вариант D из обсуждения).

Реалистичный SVG не делаем — лицензионно чистого красивого набора в стиле дизайна Формы нет. Покупка/найм иллюстратора — на бету.

## 6. XP-система

### Базовая концепция: «недельная зарплата за грамотность»

Не «копи XP вечно за объём» (плохая модель — провоцирует garbage volume).
А «получай XP за работу в правильной зоне».

### Модель начисления

```
Зона работы              XP за неделю
────────────────────────────────────────────
0 сетов                   0
1 до MEV-1               +5
MEV до MAV (оптимум)     +20
MAV до MRV (много но ок) +20  (потолок)
> MRV (перегруз)         0
```

При перегрузе — **0 XP, не минус**. Никаких отрицательных скачков уровня. Это унизительно.

Множители за прогрессию (от ProgressionEngine):
- Просто отработал зону → ×1.0 (база)
- Закрыл диапазон в одном упражнении → ×1.2
- Поднял рабочий вес в одном упражнении → ×1.5
- Несколько подъёмов → ×2.0
- Регресс/деload по плану → ×1.0 (без штрафа)
- DELOAD-неделя по плану → ×1.5 базовый XP (поощрение восстановления)

### Дробное начисление с финализацией

XP идёт **в процессе недели** как «предварительный»:
- После каждой тренировки добавляем сеты к недельному счётчику
- Показываем «грудь 9/14 сетов, гарантированно +20 XP при сохранении зоны»
- В воскресенье ночью — финализация по итоговым числам

Если в воскресенье оказался в перегрузе — XP за неделю = 0 (но не минус).

### Шкала уровней

Удвоение XP до следующего уровня:

| Уровень | XP до след. | Когда достижимо |
|---|---|---|
| Lv.1 → Lv.2 | 20 | 1 хорошая неделя |
| Lv.2 → Lv.3 | 40 | 2 недели |
| Lv.3 → Lv.4 | 80 | месяц |
| Lv.4 → Lv.5 | 160 | 2 месяца |
| Lv.5 → Lv.6 | 320 | 3-4 месяца |
| Lv.6 → Lv.7 | 640 | 8 месяцев |
| Lv.7 → Lv.8 | 1280 | год+ |
| Lv.8 → Lv.10 | ... | 3-4 года |

Lv.10 = терминальная стадия для альфы. Расширяем потом.

### Связь с фигурой

Уровень → толщина соответствующей зоны на фигуре:
- Lv.1-2: тонкая
- Lv.3-4: средняя
- Lv.5-6: широкая
- Lv.7+: массивная

Плавная интерполяция через параметры в Compose Canvas. Каждый уровень даёт +небольшое утолщение.

### UI level-up

Оверлей-карточка после тренировки, перед AI-разбором. Тап для дисмисса. Не агрессивный.

```
🎉 Level Up
Грудь — Lv.5
+15 XP за +2.5 кг в жиме лёжа
```

## 7. Этическая защита от гриnd

**Архитектурно невозможно нагриndить уровень:**

1. **XP cap по MAV/MRV.** Лишние сеты после MAV не дают XP. После MRV — 0 за неделю. Гриnd физически бесполезен.

2. **Видимость причин XP.** Юзер видит «+20 за поднятый вес», «+1 за каждый сет в зоне» — формирует понимание что важнее.

3. **AI-разбор как сторож.** Если перевыполнил MRV в 2x — AI говорит «снизь объём, ты переутомляешь грудь». Не как штраф, а как тренерский совет.

4. **XP за восстановление.** DELOAD-неделя даёт ×1.5 базовый XP. Поощряем цикл, а не «больше = больше».

### Чего НЕ делаем

- Стрики (5 тренировок подряд) — давят на тренировку когда не надо
- Лидерборды — нездоровая конкуренция
- Достижения за общий объём — поощряют гриnd
- Декей XP при пропуске — демотивирует после болезни/командировки

## 8. Плато — детекция и преодоление

### Детекция

Плато на **упражнении**, не на группе. Грубо «грудь в плато» — некорректно. Конкретно «жим лёжа в плато» — точно.

Условие: ProgressionEngine не выдавал INCREASE_WEIGHT и INCREASE_REPS **2-3 недели подряд** для конкретного упражнения. Это уже почти есть в движке через STAGNATION ветку матрицы.

### Стратегии преодоления

При детекции плато AI в разборе предлагает на выбор:

1. **Деload-неделя** — снизить вес на 15%, отдохнуть голову
2. **Сменить диапазон повторов** — был 6-8, переключаемся на 8-12 на 4 недели
3. **Сменить упражнение** — жим лёжа → жим под углом на 6-8 недель
4. **Проверить базовые факторы** — сон, еда, стресс (часто плато не от тренировок)

Юзер выбирает → ProgressionEngine корректирует параметры → плато снимается.

### Награда за преодоление

При первом INCREASE_WEIGHT после применения стратегии — **бонус +50 XP** в эту группу. Награда за результат, не за намерение. Засчитывается через 2 недели после применения, когда снова пошёл прогресс.

### Где показывать плато

- В Review (AI-разборе) — главное место, AI проактивно поднимает тему
- На WorkoutDetailScreen — рядом с упражнением badge «плато 3 недели»
- На экране Баланса — список «упражнения в плато»
- Уведомление — после 3-й подряд тренировки без прогресса

## 9. Перерывы (отдельный вопрос — обсуждали частично)

### Принципы

- **Уровни никогда не сбрасываются.** Они навсегда.
- **Никаких «ты потерял Lv.3»** — это унизительно и физиологически неверно.
- **При перерыве — мягкое возвращение**, не «начни сначала».
- **Приложение распознаёт перерыв** через `lastWorkedAt` и адаптируется.

### По длительности

- **1 неделя** — норма, никаких сигналов
- **2-4 недели** — `rust`-индикатор (бледность зоны на фигуре), AI-разбор предлагает начать с 80% весов на 2 недели восстановления
- **1-3 месяца** — мягкий перезапуск, рабочие веса автоматически снижаются с подтверждением: «У тебя был перерыв N недель. Рекомендую снизить веса на 15% на ближайшие 2 недели. Применить?»
- **6+ месяцев** — предложить новую программу через AI или повторно загрузить preset, активный прогресс с минимума

### Visual

Тонкая бледность зоны мышц на фигуре при `daysSinceLastWorkout > 14`. Без кричащих badge — органично. Через неделю работы яркость возвращается.
