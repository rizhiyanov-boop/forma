# Форма

Android-приложение для тренировок с AI-генерацией программы через OpenAI.

Это **альфа-версия этапа 1** — скелет проекта, готовая генерация программы через AI, журнал тренировок с подходами. Аналитика, графики и еженедельный AI-разбор идут следующим этапом.

## Что уже работает

- **Онбординг** — 6 шагов: цель, уровень, дни в неделю + длительность, оборудование, антропометрия, сводка
- **Генерация программы** — OpenAI `gpt-4o-mini` с Structured Outputs, возвращает валидный JSON по схеме
- **Главный экран** — сегодняшняя тренировка, статистика недели (streak, объём, кол-во тренировок), список тренировок программы (всё кликабельное)
- **Экран дня тренировки** — список упражнений с метриками, тап раскрывает детали, кнопка «Заменить» на каждом упражнении
- **Экран упражнения** — название, группа мышц, инструкция по выполнению, подсказки тренера, оборудование, плейсхолдер под видео
- **Замена упражнения через AI** — bottom sheet с опциональной причиной (боль, нет оборудования, скучно, своё), 2-3 варианта от AI с обоснованием, выбор «Только сегодня» или «Заменить навсегда»
- **Экран тренировки** — журнал подходов с ручным вводом веса и повторов, чекбокс завершения с haptic feedback, таймер отдыха с обратным отсчётом, счётчик суммарного объёма в кг
- **Хранение** — локальная Room-база, все модели готовы к синку с бэкендом (поля `isDirty`/`syncedAt` заложены заранее)

> ⚠️ **Миграция БД:** при первом запуске обновлённой версии Room сотрёт старую базу (версия БД 1 → 2). Тебе нужно будет пройти онбординг заново. На альфе это нормально.

## Стек

- Kotlin 2.0.21 + Jetpack Compose (Material 3)
- Clean Architecture: `data` / `domain` / `presentation`
- Hilt для DI
- Room 2.6 для локальной БД
- Retrofit + OkHttp + Kotlinx Serialization для OpenAI API
- Navigation Compose
- Vico (подключена, будет использоваться на этапе 2 для графиков)
- WorkManager (подключен, на этапе 2 для еженедельного AI-разбора)

## Как запустить

### 1. Подготовь окружение

- Android Studio **Ladybug (2024.2.1)** или новее
- JDK **17** (Android Studio обычно идёт со встроенным)
- Android SDK: Platform 35, Build Tools 35+

### 2. Клонируй и открой проект

Распакуй архив или склонируй репозиторий, затем открой папку `forma` в Android Studio. Gradle Sync запустится автоматически.

### 3. Добавь OpenAI API ключ

Скопируй файл-шаблон и впиши свой ключ:

```bash
cp local.properties.example local.properties
```

Открой `local.properties` и замени строку на свою:

```properties
sdk.dir=/путь/к/твоему/Android/Sdk
OPENAI_API_KEY=sk-твой_реальный_ключ
```

Ключ можно получить на https://platform.openai.com/api-keys. Файл `local.properties` в `.gitignore` и не попадёт в репозиторий.

### 4. Скачай Gradle Wrapper JAR

В архив не включён `gradle/wrapper/gradle-wrapper.jar` (бинарь ~60 КБ). Android Studio сгенерирует его автоматически при первом открытии проекта. Если этого не произошло, выполни в терминале:

```bash
gradle wrapper --gradle-version 8.9
```

(Требуется установленный Gradle.) Либо скопируй `gradle-wrapper.jar` из любого другого Android-проекта в `gradle/wrapper/`.

### 5. Собери и запусти

В Android Studio выбери устройство или эмулятор (API 26+) и нажми **Run** (▶︎). Первая сборка займёт несколько минут — скачиваются зависимости.

Приложение откроется на экране онбординга. Пройди 6 шагов, нажми «Создать программу» — через 5-15 секунд AI сгенерирует недельный план, и ты попадёшь на главный экран.

## Структура проекта

```
app/src/main/java/com/forma/app/
├── FormaApp.kt                  # Application с @HiltAndroidApp
├── MainActivity.kt              # Точка входа, ставит Compose
├── data/
│   ├── local/                   # Room: entities, DAO, database
│   ├── remote/                  # OpenAI API: Retrofit, DTO, схемы
│   ├── mapper/                  # Domain <-> Entity <-> DTO
│   └── repository/              # Реализации репозиториев
├── domain/
│   ├── model/                   # Чистые модели: UserProfile, Program, WorkoutSession
│   └── repository/              # Интерфейсы репозиториев
├── di/                          # Hilt модули
└── presentation/
    ├── theme/                   # Цвета, типографика, FormaTheme
    ├── components/              # Переиспользуемые компоненты
    ├── navigation/              # NavGraph + маршруты
    └── screens/
        ├── onboarding/          # 6-шаговый онбординг
        ├── home/                # Главный экран
        └── workout/             # Экран текущей тренировки
```

## Лицензия и ключ

API-ключ вшивается в `BuildConfig` через `local.properties` — это подходит **только для локальной альфы**. Перед публичным релизом ключ нужно вынести на бэкенд (прокси-сервер) или заставить пользователя вводить свой в настройках. Текущая архитектура (Repository pattern) позволяет сделать это без переделки UI-слоя.
